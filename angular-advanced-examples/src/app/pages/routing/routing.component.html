Routing

Ways of Route changing

Router.navigate

The router.navigate method is used to navigate to a different route programmatically. It takes an array of route segments as its first argument and an optional second argument for additional options. The code in this example to navigate to the 'animation' route happens in the component's TypeScript file, typically in a method that is triggered by a user action (like a button click).

Router.navigateByUrl

The router.navigateByUrl method is used to navigate to a different route using a URL string. It takes a single argument, which is the URL to navigate to. This method is useful when you want to navigate to a specific URL without constructing an array of route segments.

RouterLink

The routerLink directive is used in the template to create a link to a different route. It takes an array of route segments as its value. When the user clicks on the link, the router will navigate to the specified route. You have to import the RouterModule in your module or in the @ decorator to use the routerLink directive. Also you can add parameters to the route by using the routerLink directive with an array of segments. For example, [routerLink]="['/user', userId]". Also you can use the routerLinkActive directive to add a class to the link when the route is active. This is useful for styling the active link or indicating to the user which route they are currently on. And there is the queryParams directive to add query parameters to the URL like routerLink="/user" [queryParams]="{ id: userId }".
